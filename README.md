# Email Spam Detector

A full-stack web application that classifies emails as **spam** or **ham** using the OpenAI API, built with Flask and PostgreSQL.

## Features

- 🔐 User registration and login (hashed passwords)
- ✉️ Email composition with pre-send **spam checking**
- 🤖 LLM-powered classification: label, confidence score, reasons, risk flags, and a safe summary
- 📥 Inbox with per-email spam metadata
- 🛡️ Open-redirect protection and prompt-injection defence

## Tech Stack

| Layer       | Technology                          |
|-------------|-------------------------------------|
| Back-end    | Python 3.11 · Flask 3.1             |
| Database    | PostgreSQL · SQLAlchemy 2.0         |
| Auth        | Flask-Login 0.6 · Werkzeug 3.1      |
| AI / ML     | OpenAI API (`gpt-4.1-mini`)         |
| Front-end   | Jinja2 templates · HTML5 / CSS3     |

## Quick Start

```bash
# 1. Clone and install dependencies
git clone https://github.com/afathima-1080/spamdetector.git
cd spamdetector
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. Configure environment variables
cp .env.example .env
# Edit .env: set SECRET_KEY, DATABASE_URL, OPENAI_API_KEY

# 3. Create the PostgreSQL database
createdb spamdetector

# 4. Run the app (tables are created automatically)
python app.py
```

Open <http://127.0.0.1:5000> in your browser.

## Running Tests

```bash
DATABASE_URL=sqlite:///test.db pytest
```

No OpenAI API key is needed for the test suite — the `openai` module is mocked.

## Generating the Documentation PDF

The PDF is produced automatically by a GitHub Actions workflow whenever `docs/*.md` files change on the target branch.

**To run it manually:**

1. Go to **Actions** → **Build Documentation PDF** in this repository.
2. Click **Run workflow** → select the branch → **Run workflow**.
3. The workflow installs Pandoc + XeLaTeX, converts `docs/Email-Spam-Detector-Documentation-2026.md` to PDF, archives the old PDF to `docs/old/`, and commits `docs/Email-Spam-Detector-Documentation-2026.pdf` back to the branch.

Workflow file: [`.github/workflows/build-docs-pdf.yml`](.github/workflows/build-docs-pdf.yml)
